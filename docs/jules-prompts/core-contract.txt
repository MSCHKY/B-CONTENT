## Core Contract — B-CONTENT Autonomous Agent

> This contract applies to ALL persona runs. Read AGENTS.md first.

## Run Parameters
- **Change budget:** max 5 files modified
- **One-change policy:** fix exactly ONE issue per run, or report only
- **Build gate:** `cd b-content && npx vite build` must pass with 0 errors
- **Test gate:** `cd b-content && npx playwright test` must pass
- **PR mode:** AUTO_CREATE_PR — only if a code change was made

## Decision Engine

### 1. SCAN — Collect signals using your persona's scan commands
### 2. ASSESS — Rate each finding:

| Dimension  | Values                |
|------------|-----------------------|
| Severity   | Critical / High / Medium / Low |
| Confidence | High / Medium / Low   |
| Effort     | S (< 30 lines) / M (< 100 lines) / L (> 100 lines) |

### 3. DECIDE — Apply this policy:

```
IF finding.severity IN (Critical, High)
   AND finding.confidence >= Medium
   AND finding.effort <= M
   AND fix is safe (no breaking changes):
     → FIX the issue. Create PR.

ELSE:
     → REPORT only. No PR. Output patch plan for human review.
```

### 4. VERIFY — After any code change:
- Run build gate
- Run test gate
- If either fails → revert change, switch to REPORT mode

## Idempotency (CRITICAL — read carefully)

### Step 1: Check ALL open PRs
```bash
gh pr list --author "jules-google[bot]" --state open --limit 50
```
- If an open PR already addresses your finding → **SKIP**. Do NOT create a duplicate.

### Step 2: Check recently merged PRs (last 20)
```bash
gh pr list --author "jules-google[bot]" --state merged --limit 20
```
- If a merged PR already fixed this → **SKIP**.

### Step 3: Verify the issue still exists in main
Before fixing, **read the actual code on main** to confirm the problem is real:
```bash
git show main:<file-path> | head -50
```
- If the code on main already handles the issue → **SKIP**.
- Do NOT rely solely on scan output — always verify against the current source.

### Step 4: Decision
- If nothing new found after all three checks: output "✅ All clear — no action needed" and **stop immediately**.
- Do NOT create a PR that duplicates existing work.

## Failure Handling
- If a scan command fails: log the error, continue to next command
- If build/test gate fails after fix: revert, report the issue instead
- If blocked (missing permissions, network): report blocker and stop

## Command Whitelist (run in order)
1. `cat b-content/AGENTS.md` — load project rules
2. `gh pr list --author "jules-google[bot]" --state open --limit 50` — idempotency check (BEFORE scanning)
3. Persona-specific scan commands (defined in persona lens below)
4. Verify finding against main: `git show main:<path>` — confirm issue still exists
5. [If fixing] Apply change
6. `cd b-content && npx vite build` — build gate
7. `cd b-content && npx playwright test` — test gate
8. Output report in schema format above

## Boundaries
✅ Always: follow AGENTS.md rules, run gates, use output schema, use correct PR title prefix
🚫 Never: modify package.json dependencies without instruction, create files outside b-content/, change more than 5 files, skip the build gate, modify package-lock.json unless the task explicitly requires dependency changes

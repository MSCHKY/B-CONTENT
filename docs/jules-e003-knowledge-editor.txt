## Goal
Extend the existing read-only Knowledge Base Viewer (`b-content/src/app/components/knowledge/knowledge-viewer.tsx`) with CRUD capabilities for Topics and Quotes. Currently, the knowledge data lives in static JSON files under `b-content/src/data/`. The goal is to:

1. **Migrate the data source** from static JSON imports to D1 database storage
2. **Add CRUD API routes** in the worker for topics and quotes
3. **Add editor UI** in the Knowledge Viewer component

## Architecture Decision: D1 Migration

### New D1 Tables (add to `b-content/db/schema.sql`)
```sql
CREATE TABLE IF NOT EXISTS kb_topics (
  id TEXT PRIMARY KEY,
  label TEXT NOT NULL,
  kernbotschaft TEXT NOT NULL,
  facts TEXT NOT NULL DEFAULT '[]',       -- JSON array of strings
  keywords TEXT NOT NULL DEFAULT '[]',    -- JSON array of strings
  instances TEXT NOT NULL DEFAULT '{}',   -- JSON object {alex: 1-3, ablas: 1-3, bwg: 1-3}
  created_at TEXT DEFAULT (datetime('now')),
  updated_at TEXT DEFAULT (datetime('now'))
);

CREATE TABLE IF NOT EXISTS kb_quotes (
  id TEXT PRIMARY KEY,
  author TEXT NOT NULL,           -- 'alex' | 'ablas' | 'fichtel' | 'pasini' | 'general'
  author_name TEXT NOT NULL,      -- Display name e.g. 'Jürgen Alex'
  content TEXT NOT NULL,
  topics TEXT NOT NULL DEFAULT '[]',  -- JSON array of topic IDs
  emotion TEXT NOT NULL DEFAULT '',
  context TEXT,
  created_at TEXT DEFAULT (datetime('now')),
  updated_at TEXT DEFAULT (datetime('now'))
);
```

### Seed Script
Create `b-content/db/seed-knowledge.sql` that INSERTs all existing data from `b-content/src/data/topics/topic-fields.json` (13 topics) and `b-content/src/data/quotes/quotes.json` (29 quotes across 5 author groups) into the new tables. Use the existing IDs and data exactly as they are.

### API Routes (extend `b-content/worker/routes/knowledge.ts`)

Keep the existing GET endpoints but switch them to read from D1 instead of static imports. Add:

```
GET    /api/knowledge/topics          → List all topics from D1
GET    /api/knowledge/topics/:id      → Get single topic
POST   /api/knowledge/topics          → Create topic (validate: id, label, kernbotschaft required)
PUT    /api/knowledge/topics/:id      → Update topic
DELETE /api/knowledge/topics/:id      → Delete topic

GET    /api/knowledge/quotes          → List all quotes (grouped by author)
GET    /api/knowledge/quotes?author=X → Filter by author
POST   /api/knowledge/quotes          → Create quote (validate: id, author, author_name, content required)
PUT    /api/knowledge/quotes/:id      → Update quote
DELETE /api/knowledge/quotes/:id      → Delete quote
```

Use the existing validation patterns from `b-content/worker/services/validation.ts`.
Use the existing error format: `{ error: string, code?: string }`.
All D1 queries MUST use parameterized statements (no string concatenation).

### Frontend: Knowledge Editor UI

Extend the existing `KnowledgeViewer` component with edit capabilities:

1. **Add/Edit Topic Modal**: Form with fields for label, kernbotschaft, facts (dynamic list), keywords (dynamic list), instance weights (3 dropdowns for alex/ablas/bwg each 1-3)
2. **Add/Edit Quote Modal**: Form with fields for content, author (dropdown), emotion, context (optional), topic tags (multi-select)
3. **Delete Confirmation**: Simple confirm dialog before delete
4. **Action Buttons**: Add "+" button at the top of Topics/Quotes tabs, Edit/Delete icons on each item (use Lucide: Plus, Pencil, Trash2)
5. **State**: Fetch data from API instead of static imports. Use React state (useState + useEffect) for data fetching. No new Zustand store needed.

### UI Design Requirements
- Use existing glass-card styling and Tailwind classes
- Modals should use a simple overlay pattern (fixed position, backdrop-blur)
- Match the existing visual design (glassmorphism, gradient accents)
- German UI labels: "Neues Thema", "Neues Zitat", "Bearbeiten", "Löschen", "Speichern", "Abbrechen"
- Use Lucide React icons, NEVER emoji in UI
- Keep the existing tab structure (Topics / Quotes / Rules)
- Rules tab stays read-only (no CRUD needed)
